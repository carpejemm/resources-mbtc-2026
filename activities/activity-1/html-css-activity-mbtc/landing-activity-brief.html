Landing Page Project

Build a fully responsive landing page for a fictional tech app using HTML & CSS
Score:
/ 50

Project: You will build a landing page for Syncly — a fictional project management app. The page must include the sections listed below and meet all 20 technical requirements. Open the starter files landing-starter.html and landing-starter.css, read the TODO comments, and implement each requirement.

Files to submit: landing-starter.html and landing-starter.css (rename to your name before submitting).  Each requirement is worth 2.5 pts (Total: 50 pts).

Section 1  ·  Navigation Bar Requirements 1–3
  • 1 Use a semantic <nav> element. Inside it, place the logo text on the left and three navigation links on the right. Use Flexbox with justify-content: space-between to push them apart. HTML + CSS
  • 2 Make the navbar stay fixed at the top while the user scrolls using position: sticky and top: 0. Add a subtle bottom border or box-shadow. CSS
  • 3 Add a smooth color-change transition to all nav links on :hover (transition duration: 0.3s ease). CSS
Section 2  ·  Hero Section Requirements 4–7
  • 4 Use Flexbox to center the hero content both horizontally and vertically. The hero section should have a minimum height of 80vh. CSS
  • 5 Write a @keyframes animation called fadeSlideIn that starts with opacity: 0 and transform: translateY(30px), ending at opacity: 1 and transform: translateY(0). Apply it to the hero heading. CSS
  • 6 Declare at least 3 CSS Custom Properties on :root — a primary color, an accent color, and a text color — and use them throughout the page. CSS
  • 7 Style the primary CTA button with border-radius: 8px, a box-shadow, and a :hover transition that shifts the background color. CSS
Section 3  ·  Features Grid Requirements 8–10
  • 8 Display the three feature cards in a 3-column CSS Grid using repeat(3, 1fr) with a 24px gap. Each card must have a heading, icon (emoji is fine), and short description. HTML + CSS
  • 9 Give each feature card a border-radius, a subtle box-shadow, and a border. CSS
  • 10 Add a :hover effect to each card using transform: translateY(-6px) and a transition so the card smoothly "lifts up". CSS
Section 4  ·  How It Works Requirements 11–13
  • 11 Use an <ol> with list-style: none. Add custom step numbers using a CSS counter and the ::before pseudo-element (display as a circle with the count inside). HTML + CSS
  • 12 Use the :nth-child(even) selector to give alternating list items a light background color that differs from the odd items. CSS
  • 13 Set the width of the "How It Works" content wrapper using calc() — it should be 100% minus 48px for padding. CSS
Section 5  ·  Testimonials Requirements 14–15
  • 14 Place two testimonials side-by-side in a 2-column CSS Grid. Use a <blockquote> element for each testimonial with a <cite> for the author name. HTML + CSS
  • 15 Add a large decorative opening quote mark (") to each <blockquote> using the ::before pseudo-element (content, font-size, color — no extra HTML element allowed). CSS
Section 6  ·  Contact Form Requirements 16–17
  • 16 Build a contact form with four fields: text (name), email, textarea (message), and a submit button. Each field must have a <label> and proper name attribute. HTML
  • 17 Style form inputs using the :focus pseudo-class: change the border-color to your primary color and add a soft box-shadow glow effect on focus. CSS
Section 7  ·  Footer & Semantic HTML Requirements 18–19
  • 18 Build the footer with 3 columns (Links, Social, Legal) using Flexbox with justify-content: space-between. Wrap the whole footer in a <footer> element. HTML + CSS
  • 19 Ensure the full page uses correct HTML5 semantic elements: <header>, <nav>, <main>, <section>, <article>, <footer>. Add an aria-label attribute to at least two sections. HTML
Section 8  ·  Responsive Design Requirement 20
  • 20 Write a mobile-first media query at max-width: 768px that: collapses the features grid to 1 column, collapses the testimonials to 1 column, stacks the navbar links vertically, and increases the hero heading font size for readability. CSS

Tips before you start:
  • Open landing-starter.html and landing-starter.css side-by-side. All TODO comments tell you exactly where to write your code.
  • Use browser DevTools (F12) to inspect and debug your layout in real time.
  • Test your responsive design by dragging the browser window smaller, or use DevTools device toolbar (Ctrl+Shift+M / Cmd+Shift+M).
  • You can use any color palette you like — just define it through CSS Custom Properties.
